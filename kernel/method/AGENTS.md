# GameSpec 稳定核心契约

本文件是每个项目安装后的 GameSpec 入口。

它必须在项目只安装默认 `stable-core` profile 时仍然成立。不要假设可选 beta 文件、运行时适配器、角色、技能、模板或工作流定义已经存在。

## 已安装契约

默认稳定核心只保证存在以下文件：

- `config.yaml`
- `AGENTS.md`
- `install-surface.json`

安装面清单是以下信息的真相源：

- 已安装契约版本
- 默认安装 profile
- 可选安装 profile
- 受保护的项目真相边界
- 兼容性与写入策略

如果不确定当前项目安装了哪些 GameSpec 能力，先读取 `install-surface.json`，再做判断。

## 项目真相边界

项目设计真相属于目标项目，不属于 GameSpec 产品包。

规则：

- 产品安装命令不得覆盖项目真相。
- 产品同步命令必须经过显式审查和批准。
- 已存在且内容不同的产品托管文件必须阻塞自动替换。
- `stable-core` 项目缺少可选 beta 文件不是错误。

## 稳定核心行为

在稳定核心安装中工作的 agent 应当：

1. 只把本文件、`config.yaml` 和 `install-surface.json` 视为确定存在的 GameSpec 文件。
2. 保留用户意图，并把决策记录在项目自有文档中。
3. 依赖可选 beta 安装面之前，先确认用户已经显式选择对应安装 profile，或确认相关文件确实存在。
4. 将生成或修改的项目内容与产品托管安装文件分开。
5. 对任何可能影响项目文件的动作，优先做只读检查和计划。

## 可选安装面

其他 GameSpec profile 可以安装额外的设计协议、角色提示、技能、模板、工作流和运行时适配器。

这些文件都是可选能力。只有在确认它们已经实际安装，或用户明确选择了提供这些能力的安装 profile 后，才可以依赖它们。

## 版本信号

GameSpec 将版本信号分开管理：

- `install-surface.json` 描述已安装的核心与运行时契约。
- `config.yaml` 描述项目级默认值与本地偏好。
- package version 描述分发产物版本。
- 运行时适配器元数据描述宿主侧兼容性。

不要把这些信号合并成一个版本号。
