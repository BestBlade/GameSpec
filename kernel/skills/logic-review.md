## 概述

逻辑审查技能负责对设计文档进行深度逻辑分析，检测因果关系是否闭环、定义是否一致、边界条件是否覆盖、以及文档内部或跨文档之间是否存在矛盾。输出结构化的审查报告，按严重程度排序并给出修复建议。

## 步骤

1. **因果闭环检查** — 追踪文档中每个"如果...则..."关系，检查是否存在未定义的前因、缺失的后果、或循环依赖
2. **定义一致性检查** — 检查同一概念在文档中的多处定义是否一致（名称、数值、行为描述）
3. **边界条件检查** — 识别数值和逻辑的边界情况（极值、零值、并发、空集），检查是否有明确处理
4. **矛盾检测** — 检测文档内部的逻辑矛盾（如：A处说X>Y，B处说Y>X）和跨文档矛盾
5. **生成报告** — 汇总所有发现，按 Critical/Major/Minor 分级排列

## 输出格式

```markdown
# 逻辑审查报告

## 审查概述
- **文档**: [文档名称]
- **审查时间**: [时间]
- **问题统计**: Critical: [n] | Major: [n] | Minor: [n]

## 问题列表

### [LR-001] [问题标题]
- **级别**: Critical / Major / Minor
- **类型**: 因果缺失 / 定义冲突 / 边界缺失 / 逻辑矛盾
- **位置**: [文档章节/行号]
- **描述**: [问题描述]
- **影响**: [可能造成的后果]
- **修复建议**: [建议]

## 修复优先级
1. [最高优先级问题]
2. ...
```

## 示例

**输入**:
- document: "战斗系统设计文档，其中描述'护盾吸收所有伤害'，同时在另一处描述'穿透伤害无视一切防御'"

**输出**:

```markdown
# 逻辑审查报告

## 问题列表

### [LR-001] 护盾与穿透伤害的矛盾
- **级别**: Critical
- **类型**: 逻辑矛盾
- **位置**: 第3章「护盾机制」 vs 第5章「穿透伤害」
- **描述**: 护盾定义为"吸收所有伤害"，穿透伤害定义为"无视一切防御"。两者交互时行为未定义。
- **影响**: 实现时必然产生歧义，不同开发者可能有不同理解
- **修复建议**: 明确优先级关系，如"穿透伤害绕过护盾"或"护盾对穿透伤害仅吸收50%"
```
