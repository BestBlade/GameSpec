## 概述

`drift-rollup` 是 M8（局部信号到战略）的核心技能。

10 个变更过去了，3 个带着同样的债务——每个单独看都合理，合在一起是模式。M8 强制这种跨变更视角。

## 双模式

### 战略汇总 (strategic rollup)

**数据源**: ≥3 个完整归档的 ARCHIVE_SUMMARY 文件
**阈值**: ≥3 个相同漂移模式实例
**输出**: 可操作建议——更新模板、修改工作流、修订宪法、人类决策记录

### 漂移快照 (drift snapshot)

**数据源**: 所有项目的审查报告 + DISP_ 辩论记录（含活跃项目）
**阈值**: 无（报告所有新兴模式，不设阈值）
**输出**: 新兴模式标记——不做战略主张，仅做风险提示

## 步骤

1. **收集数据** — 根据模式读取对应数据源
2. **分类模式** — 重复的考古未知项、重复的 fallback/债务接受、重复的 INTT_ 范围蔓延事件、重复的辩论失败场景
3. **按出现次数排列** — 不自动将频率等同于优先级
4. **提出建议** — 模板更新 / 工作流更新 / 宪法修订 / 人工决策记录
5. **数据完整度说明** — 多少项目贡献了数据、处于什么阶段、数据完整到什么程度

## 输出

```markdown

## 1. 数据完整度
| 项目 | 阶段 | 归档数 | 审查报告数 | 数据完整度 |
| [项目名] | [阶段] | N | N | 完整 / 部分 |

## 2. 已识别模式
| 模式 | 出现次数 | 受影响项目 | 首次出现 | 最近出现 |
| [模式描述] | N | [项目列表] | YYYY-MM | YYYY-MM |

## 3. 被挑战的假设
[这些模式暗示哪些更大的假设可能需要修正？]

## 4. 建议更改
[模板 / 工作流 / 宪法 / 流程层面的建议]

## 5. 需人工决策
[需要 L1 判断的值判断]
```

## 约束

- 漂移快照不产出战略建议（避免从不足数据做出主张）
- 数据完整度章节必须明确哪些结论基于完整数据、哪些基于部分数据
- 单次偶发事件不触发战略建议（战略汇总模式）
