## 概述

规则撰写技能负责将策划的自然语言需求描述转化为结构化、可执行的原子规则文档。每条规则应当是最小可独立验证的逻辑单元，包含明确的触发条件、执行逻辑和异常处理。所有数值参数使用 `{{VAR}}` 变量语法进行抽象，确保规则与具体数值解耦。

## 步骤

1. **拆解需求为原子操作**
   - 阅读自然语言需求，识别其中包含的独立逻辑单元
   - 每个原子操作应满足"单一触发、单一效果"原则
   - 若需求包含多个原子操作，为每个操作生成独立规则文档
   - 标注规则之间的依赖关系（如有）

2. **定义触发条件（Trigger）**
   - 明确规则的触发事件（如：玩家进入区域、血量低于阈值）
   - 定义前置条件（Precondition）：规则生效所需的系统状态
   - 使用逻辑运算符连接多个条件：`AND`、`OR`、`NOT`
   - 示例格式：
     ```
     触发事件: 玩家攻击命中目标
     前置条件: 玩家持有武器 AND 目标处于可受击状态
     ```

3. **定义执行逻辑（Execution）**
   - 按执行顺序列出操作步骤
   - 每步操作使用"主语 + 动词 + 宾语"句式
   - 数值参数使用 `{{VAR_NAME}}` 变量替代硬编码数字
   - 标注操作的同步/异步属性
   - 示例格式：
     ```
     1. 计算伤害值 = 攻击力 × {{DAMAGE_MULTIPLIER}} - 目标防御力
     2. 目标生命值减少【伤害值】
     3. 触发受击动画（异步）
     ```

4. **定义异常处理（Exception）**
   - 列举可能的异常场景（如：目标已死亡、数值溢出、并发冲突）
   - 为每个异常定义处理策略：忽略 / 中断 / 回滚 / 降级
   - 定义边界条件的处理方式（如：数值下限为 0）

5. **使用 `{{VAR}}` 变量抽象数值**
   - 识别规则中的所有硬编码数值
   - 为每个数值定义变量名，遵循 `UPPER_SNAKE_CASE` 命名规范
   - 在规则末尾的变量定义表中列出所有变量及其默认值、取值范围、单位
   - 示例：
     ```
     | 变量名 | 默认值 | 范围 | 单位 | 说明 |
     |--------|--------|------|------|------|
     | {{DAMAGE_MULTIPLIER}} | 1.0 | 0.1-10.0 | 倍率 | 伤害倍率系数 |
     ```

## 示例

### 示例1：暴击伤害规则

**输入**:
- requirement: "玩家攻击时有一定概率触发暴击，暴击伤害为普通伤害的1.5倍"
- system_id: combat
- type: atomic_rule

**输出**:

```markdown
# 规则：暴击伤害计算

## 元数据
- **规则ID**: combat-atomic_rule-001
- **所属系统**: combat
- **类型**: atomic_rule
- **版本**: 1.0
- **依赖规则**: combat-atomic_rule-base_damage

## 触发条件
- **触发事件**: 玩家攻击命中目标
- **前置条件**: 攻击判定为命中 AND 目标处于可受击状态

## 执行逻辑
1. 生成随机数 R（0-100）
2. 若 R <= {{CRIT_CHANCE}}，标记本次攻击为暴击
3. 若为暴击，最终伤害 = 基础伤害 × {{CRIT_MULTIPLIER}}
4. 若非暴击，最终伤害 = 基础伤害

## 异常处理
| 异常场景 | 处理策略 | 说明 |
|----------|----------|------|
| 暴击率超过100% | 截断为100% | 保证暴击率上限 |
| 最终伤害为负数 | 设为0 | 伤害下限保护 |

## 变量定义
| 变量名 | 默认值 | 范围 | 单位 | 说明 |
|--------|--------|------|------|------|
| {{CRIT_CHANCE}} | 15 | 0-100 | % | 暴击概率 |
| {{CRIT_MULTIPLIER}} | 1.5 | 1.0-5.0 | 倍率 | 暴击伤害倍率 |
```

### 示例2：每日登录奖励规则

**输入**:
- requirement: "玩家每天首次登录获得金币奖励，连续登录天数越多奖励越高，7天一个循环"
- system_id: economy
- type: feature

**输出**:

```markdown
# 规则：每日登录奖励

## 元数据
- **规则ID**: economy-feature-daily_login
- **所属系统**: economy
- **类型**: feature
- **版本**: 1.0
- **依赖规则**: 无

## 触发条件
- **触发事件**: 玩家登录游戏
- **前置条件**: 当日未领取过登录奖励

## 执行逻辑
1. 获取玩家当前连续登录天数 N
2. 计算奖励档位 D = (N - 1) mod {{LOGIN_CYCLE}} + 1
3. 查询奖励表获取档位 D 对应的金币数量
4. 发放金币至玩家账户
5. 更新连续登录天数 N = N + 1

## 异常处理
| 异常场景 | 处理策略 | 说明 |
|----------|----------|------|
| 跨日判定时区差异 | 使用服务器时区 | 统一以服务器0点为日界 |
| 发放失败 | 重试3次后记录补发 | 保证玩家不丢失奖励 |
| 连续登录中断 | 重置N为1 | 重新开始计算连续天数 |

## 变量定义
| 变量名 | 默认值 | 范围 | 单位 | 说明 |
|--------|--------|------|------|------|
| {{LOGIN_CYCLE}} | 7 | 1-30 | 天 | 登录奖励循环周期 |
```
