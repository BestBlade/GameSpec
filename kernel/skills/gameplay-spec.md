## 概述

玩法规格生成技能负责将抽象的体验目标转化为具体的玩法循环设计。文档定义核心循环（Core Loop）、二级循环和元循环的结构，列出所有原子操作及其关系，并通过 MDA（Mechanics-Dynamics-Aesthetics）框架确保玩法设计与体验目标一致。

## 步骤

1. **分析体验目标** — 将体验锚点拆解为可量化的体验指标（如：紧张感、成就感、探索欲）
2. **设计核心循环** — 构建"行动→挑战→反馈→奖励"的核心玩法循环
3. **定义原子操作** — 列出玩家可执行的所有最小操作单元及其输入输出
4. **设计反馈系统** — 为每个操作定义即时反馈和延迟反馈
5. **MDA 验证** — 使用 MDA 框架验证机制（M）通过动态（D）能否产生目标美学（A）

[循环描述 + Mermaid 图]

```

## 示例

**输入**:
- experience_anchors: "紧张刺激的战斗、策略性的构筑、成长的满足感"
- design_pillars: "易学难精、每局体验不同、有意义的选择"

**输出**:

```markdown
# 玩法规格：Roguelike 卡牌战斗

## 概述
- **核心体验**: 通过策略性的卡牌构筑和紧张的回合制战斗获得成就感
- **设计支柱**: 易学难精、每局体验不同、有意义的选择

## 体验循环

### 核心循环（回合级，30秒）
```mermaid
flowchart LR
    A[观察局势] --> B[选择卡牌]
    B --> C[执行效果]
    C --> D[敌方响应]
    D --> A
```

### 二级循环（战斗级，5-10分钟）
进入战斗 → 多回合对决 → 战斗结算 → 卡牌奖励选择

### 元循环（单局，30-60分钟）
选择角色 → 多层关卡推进 → 构筑卡组 → Boss 挑战 → 结算

## 原子操作列表
| 操作ID | 操作名称 | 输入 | 输出 | 反馈 | 所属循环 |
|--------|----------|------|------|------|----------|
| ACT-01 | 打出卡牌 | 手牌选择+目标 | 效果结算 | 动画+数值变化 | 核心循环 |
| ACT-02 | 结束回合 | 确认操作 | 敌方行动 | 回合切换动画 | 核心循环 |
| ACT-03 | 选择奖励卡 | 3选1 | 卡组变化 | 卡牌入库动画 | 二级循环 |

## MDA 映射
| 机制 (M) | 动态 (D) | 美学 (A) |
|-----------|----------|----------|
| 手牌限制+费用系统 | 资源管理博弈 | 紧张感、策略深度 |
| 随机卡牌奖励 | 构筑路线分化 | 每局体验不同 |
| Boss 机制设计 | 策略针对性调整 | 成就感、挑战感 |
```
