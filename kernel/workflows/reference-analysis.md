# 参考作品拆解工作流

## 概述

本工作流将“分析别的作品”从零散讨论升级为正式流程。它的目标不是产出影评或游戏随笔, 而是:

1. 建立可复用的参考样本库;
2. 统一外部作品分析的评分标准与输出结构;
3. 把拆解结论回流到当前项目的世界观、角色、系统与叙事设计中。

## P1/P2 接线协议

- 已有项目进入本工作流前，应先由 `@game-导航` 执行 `skill: session-start`，恢复 `active.md` / `.gamespec-state.yaml` 的当前工作锚点。
- 本工作流内任何会新增或改写项目文档的阶段，在产出完成后都必须执行 `skill: post-write-validator`，清空 Blocker 后方可进入下一阶段或回流主项目。
- 在长会话压缩、任务中断、跨角色交接或准备上交 L1 前，当前执行角色必须执行 `skill: session-compact`，将 `Blockers` / `Pending Handoffs` 与 session 元数据写回项目检查点；handoff 不自动改写 `current_agent`。
- 若当前活跃文档发生切换，应同步更新项目根 `active.md`，并把该文件纳入同一次写后校验。
- P4 事实源语义同样生效：`prototype` 不得充当稳定依赖；`full` / `lean` 若依赖 `lean` 文档，正文前部必须显式写明当前口径或工作假设；`lean` 文档不得直接进入 `.ai.md -> .md`。

## M1 意图前置步骤（所有阶段通用）

本工作流的任何阶段在设计产出前，均须执行以下前置步骤：

1. L0 (`@game-导航`) 将用户原始意图（原话）传递给目标 L2。
2. L2 使用 `skill: intent-spec` 和 `TMPL_DESIGN_INTENT` 产出 `INTT_{PHASE}_{名称}.ai.md`。
3. L1 (`@game-制作人` 或 `@game-系统架构师`) 完成三确认：意图清晰性、翻译忠实性、边界合理性。
4. 三确认未全部通过 → 打回修改 INTT_；全部通过 → 阶段设计工作开始。
5. 设计过程中意图发生变更 → 停止当前阶段，更新 INTT_，重新 L1 确认。

## Phase 1: 分析目标定义

| 属性 | 值 |
|------|-----|
| **层级** | L1 |
| **参与 Agent** | @game-制作人（主导）、@game-Spec架构师 |
| **所需 Skill** | `reference-work-analysis` |

### 输入

- 当前项目要解决的问题
- 候选参考作品列表
- 评分标准文档（如有；若无, 使用项目默认评分标准）

### 工作内容

1. 明确本轮分析重点（如主角可玩性、真相拼图、制度切面、队伍共同体）
2. 选择 1-5 部最有针对性的参考作品
3. 明确本轮评分维度与“最想学的点”
4. 确认本轮使用的评分标准文档

### 输出

- 候选总览文档
- 分析任务清单

## Phase 2: 单作品拆解

| 属性 | 值 |
|------|-----|
| **层级** | L2 |
| **参与 Agent** | @game-Spec架构师（主导）、@game-玩法策划、@game-世界观 |
| **所需 Skill** | `reference-work-analysis` |

### 输入

- 选定参考作品
- 当前项目上下文
- 本轮分析重点
- 评分标准文档

### 工作内容

1. 判断作品真实顶层命题
2. 拆解主循环、公共动词、章节发动机、整合作用
3. 输出单作品拆解文档
4. 调用 `review-scoring` 或按等价标准给出总分、维度分与上下限说明

### 输出

- 单作品拆解文档（review 格式）

---

## Phase 3: 横向矩阵比较

| 属性 | 值 |
|------|-----|
| **层级** | L2 |
| **参与 Agent** | @game-Spec架构师（主导） |
| **所需 Skill** | `reference-work-analysis`、`competitor-analysis` |

### 输入

- Phase 2 的所有单作品拆解文档

### 工作内容

1. 统一维度
2. 构建横向评分矩阵
3. 校对各文档评分口径是否一致
4. 给出当前项目阶段的优先学习顺序

### 输出

- 参考作品优先级矩阵

---

## Phase 4: 项目回流

| 属性 | 值 |
|------|-----|
| **层级** | L1 |
| **参与 Agent** | @game-制作人、@game-世界观、@game-玩法策划、@game-Spec架构师 |

### 输入

- 候选总览文档
- 单作品拆解文档
- 横向评分矩阵

### 工作内容

1. 提炼“最值得学的点”
2. 明确“最不该误学的点”
3. 判断哪些结论要进入正式设计流程
4. 将结论回流到主角、世界、系统、叙事等下一阶段设计中

### 输出

- 项目回流备忘录
- 后续设计问题清单

---

## 质量门禁

- 每一部参考作品必须有单独拆解文档
- 每一轮分析必须产出横向矩阵
- 每次分析必须明确“可学点 / 禁误学点 / 对本项目的直接启示”
- 每次分析必须明确“总分 / 维度分 / 为什么不是更低 / 为什么不是更高”
- 参考分析不得停留在“作品很好”这种空泛结论上

---

## 工作流价值

将参考作品分析纳入正式流程后, GameSpec 不再把“看了什么作品”当成零散灵感来源, 而是把它们转化为:

- 可积累的样本库
- 可比较的方法论资产
- 可直接回流项目的结构性结论
